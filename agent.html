从 Kimi Code 到 Claude Code：学会一次，到处能用

你电脑上已经装好的 Kimi Code，其实和 Claude Code 是同一个程序，只是换了一个"大脑"。搞懂这一点，你就能看懂几乎所有 AI Agent 工具的原理，以后遇到新工具也能很快上手。

CONCEPT

先搞懂一个关键概念

一个是"壳"，一个是"大脑"。壳负责跟你对话、读写文件、执行命令；大脑负责思考该做什么。壳可以插上不同的大脑。

壳 · 不变的部分

Claude Code 这个程序本身

能在终端里对话、能读你的代码、能帮你写文件、能执行命令，还会在做有风险的操作前问你「要不要继续」。这一整套能力，就是「壳」。

大脑 · 可以换

Kimi 还是 Claude，只是配置不同

「Kimi Code」= Claude Code 这个壳 + 把请求发给月之暗面的 Kimi 模型。「Claude Code」= 同一个壳 + 把请求发给 Anthropic 官方的 Claude 模型。命令、用法完全一样。

!

怎么换大脑

秘密就在两个设置项：ANTHROPIC_BASE_URL（请求发到哪个网址）和 ANTHROPIC_AUTH_TOKEN（用哪个账号的钥匙）。手动改配置文件比较麻烦，所以有了 cc-switch 这个小工具，专门用来一键切换这两项设置。

GETTING STARTED

上手 Kimi Code：打好地基

1

打开终端

按 Win 键，输入「终端」或「Windows Terminal」回车打开。这是一个只能打字、没有鼠标图标的窗口——先别害怕，AI Agent 的世界大部分工作都在这里发生。

2

进入一个文件夹

比如桌面新建一个 test 文件夹，然后在终端里输入：

cd Desktop\test

这一步是在告诉 Kimi Code：「你接下来要读写的文件都在这个文件夹里」，这也是终端最核心的概念——当前所在的位置。

3

启动它

输入命令并回车：

claude

没错，即使是 Kimi Code，启动命令也是 claude——因为它本质上就是 Claude Code 这个程序，只是背后连的是 Kimi。

4

理解它的工作循环

你提需求 → 它读文件/写文件/执行命令 → 遇到有风险的操作会先问你「是否允许」→ 你确认 → 它继续，直到完成。这个「提需求 → 行动 → 确认 → 继续」的循环，就是所有 Agent 工具的通用模式。

5

三个练手小任务

  1. 让它「解释一下这个文件夹里有什么」——熟悉它怎么读取信息
  2. 让它「写一个打印 Hello World 的 Python 脚本，并运行它」——体验完整的写代码+执行流程
  3. 让它「帮我把这些文件按类型整理到子文件夹」——体验真实的自动化小任务
CC-SWITCH

cc-switch：一键切换"大脑"

一个专门管理 AI Agent 配置的小工具，Windows / Mac / Linux 都有，图形界面，不用手改配置文件。

1

下载安装

打开 github.com/farion1231/cc-switch/releases，下载最新版本的 .msi 安装包（或免安装的 zip 版本），双击安装。

2

添加 Kimi 配置

打开 CC Switch，新建一个供应商，选择 Kimi 预设（或手动填写），把已有的 Kimi API Key 填进去保存。这一步只是把你已经在用的 Kimi Code 配置「登记」进这个管理工具，方便以后一键切换。

3

一键切换

以后如果哥哥给你开通了 Claude 官方账号，或者想试试 DeepSeek、GLM 这些其它模型，直接在 CC Switch 里点一下对应的供应商，或者用系统托盘图标切换，Claude Code 支持不用重启终端就立刻生效。

GENERALIZE

举一反三：这套思路能用在哪

以后遇到任何一个新的 AI Agent 工具，都可以先问自己这三个问题。

它的「壳」能做什么？

能读写文件吗？能执行命令吗？能上网搜索吗？——这决定了它能替你完成哪些实际任务。

它的「大脑」是谁？

背后连的是哪家的模型？是否可以更换？——同样的壳换一个更强的大脑，能力可能完全不同。

它的循环是什么？

提需求 → 行动 → 确认 → 继续，几乎是所有 Agent 工具共同的模式，包括 OpenAI 的 Codex CLI、Google 的 Gemini CLI。认准这个模式，换个工具也能很快上手。

SAFETY

安全提醒，比熟练更重要

这几条一定要记住

  • API Key 就是账号密码，绝对不要截图发群里、发朋友圈，或者粘贴到不认识的网站
  • Agent 请求执行某个命令时，先读一遍它要做什么，看不懂就问它「这条命令具体会做什么」，别看到确认框就一路按回车
  • 看到「删除」「格式化」「强制覆盖」这类字眼，一定要先停下来，不确定就找哥哥确认
  • 保持怀疑：AI 说的不一定对，尤其是它说「我已经测试过了」「肯定没问题」的时候，越要自己动手验证一下
