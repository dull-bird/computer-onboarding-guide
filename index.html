写给刚拿到新电脑的你

从不会打字，到看懂 AI Agent

这台电脑是你的新起点。这份指南按时间顺序排好了顺序：先学会用电脑，再理解电脑是什么，再动手写代码，最后弄懂现在很火的 AI 到底在干什么。不用着急，一步一步来就好。

WHY

思考题：这些东西到底能帮你解决什么问题

学习最怕的不是难，是不知道学了要干嘛。下面这些都是真实会遇到的场景，先想一想自己会怎么办，再看后面对应的答案。

开学一门课要交好几次实验报告，每次都要整理一堆数据、画同样格式的图，手动做到崩溃，怎么办？

学一点 Python（对应「编程真正入门」这一步），几行代码就能批量处理数据、自动画图，改一个数字全部重跑一遍。

写文献综述要看几十篇论文摘要，一篇篇啃根本看不完，怎么快速筛出哪些值得精读？

用 AI（对应「理解 AI 与 Agent」）帮你批量摘要、归类、挑重点，一晚上能做完过去一星期的活。

手里有几百行 Excel 数据，要去重、排序、核对，一个个点鼠标要点到手抽筋，怎么办？

同样是「编程真正入门」要解决的问题——写个几十行的小脚本，几秒钟批量搞定，还不会手滑点错。

想给自己做一个课程表提醒、健身打卡、记账的小工具，完全不知道从哪下手，怎么办？

直接看「Agent 实战教程」——把需求讲清楚，让 Kimi Code 陪你把第一个真正能用的小程序写出来。

电脑用久了变卡、装的软件自己都说不清哪个是啥，出了问题只能干瞪眼，怎么办？

这是「理解计算机是什么」要打的底子——搞懂进程、内存、存储的关系，遇到问题至少知道从哪查起。

以后找实习，简历上写「熟悉 Python」的人一抓一大把，怎么让面试官一眼看出你是真的会，不是选修课划了水？

「元方法」里说的项目驱动——做出一两个真实能跑起来的小工具，比证书和课时数更有说服力。

META

元方法：比这份清单更重要的事

下面会列出具体的网站、软件、UP 主，但真正值得带走的不是这份清单本身，而是"怎么自己找到下一份清单"的能力。清单会过时，方法不会。

清单会过时，方法不会

UP 主会停更、网站会关掉、课程会下架。但"怎么判断一个学习资源好不好"这项能力，会一直有用，值得比记住具体名字更花心思。

学会自己找、自己判断

遇到不懂的概念，先试着自己搜索、问 AI，而不是等着被投喂链接。判断一个资源好不好，看评论区讨论质量、有没有配套练习、最近还更不更新，比看标题更可靠。

用 AI 加速理解，而不是替代思考

看不懂的地方，直接问 Kimi Code「用简单的话讲讲这是什么」，比死磕原文快得多。但讲完之后自己动手验证一遍——AI 说的不一定对。

能讲出来，才算真的懂

费曼学习法：学完一个概念，试着自己讲一遍或写一遍笔记。讲不清楚，说明还没真懂，回去再看一遍。

项目驱动，胜过纯听课

每学一个新概念，找个用得上的小项目练手，比刷十个教学视频都有效。看懂和做出来，是两回事。

ROADMAP

五个阶段的学习路线

1
第 1 周

先学会「操作」电脑

打字指法、文件夹与路径概念、Win11 常用快捷键、输入法与截图。每天 20-30 分钟练打字，不用赶进度，肌肉记忆最重要。

2
第 2-3 周

搭好工具环境

装好浏览器、效率工具、剪贴板和截图神器、笔记软件，养成「用工具而不是硬扛」的习惯。

3
第 1 个月

理解计算机是什么

CPU / 内存 / 硬盘的关系、操作系统在做什么、二进制与文件本质、网络基础。推荐哈佛 CS50 中文字幕版。

4
第 2 个月

编程真正入门

从 Python 零基础学起：变量、循环、函数、列表。做几个小项目：算术小游戏、词频统计、命令行小工具。

5
长期坚持

理解 AI 与 Agent

用可视化建立数学直觉，理解大模型原理，再动手用 Kimi Code 做真实小任务——整理文件、写小脚本、自动化重复劳动。想直接上手？看 Agent 实战教程。

CAUTION

新手要避开的坑

装软件时多留一个心眼

  • 不要装「电脑管家」「驱动精灵」「XX急速下载器」这类国产全家桶和第三方下载站，容易夹带流氓软件
  • Windows 11 自带的 Defender 杀毒够用，不需要额外装杀毒软件
  • 安装向导里的「附加安装」「推荐软件」默认要取消勾选，这是最容易中招的地方
  • 尽量从官网或微软商店下载软件，不要在搜索引擎的广告链接里下载
